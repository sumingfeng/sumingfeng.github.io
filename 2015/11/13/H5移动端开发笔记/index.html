H5移动端开发笔记

发表于 2015-11-13   |   分类于 F2E   |  

前端架构方案推荐：

  • RequireJS+Backbone+Zepto+Template（灵活的MVC架构方案）
  • RequireJS+Route+template+(可选：jQuery、Zepto、querySelecter……)
  • jQuery+template（流水业务）
  • querySelecter + template（流水业务）

UI选型：

  • Android Material Design：Materialize、Material UI
  • IOS：SUI Mobile
  • 跨平台响应式：Foundation、Bootstrap
  • 本地套壳式：jQueryMobile

手势库，主要应对复杂的多点触控交互：

  • 目前测试不卡的：HammmerJS

基础知识

1.viewport 如：

<meta name="viewport" content="width=device-width,
initial-scale=1.0,user-scalable=0">

表示页面的宽度为设备宽度、初始缩放比例为1，并且禁止用户缩放网页。
device-width为设备的物理宽度，width为页面的宽度

2.如何自定义手势，涉及到基本的触控事件知识：
　http://wf.uisdc.com/cn/input/touch-input/touchevents/

3.CSS3 Media queries 如：

<style>

    @media (min-width:768px) and (max-width:959px){

        /*表示页面宽度大于768px小于959px的时候执CSS代码*/
    } 

    @media (max-width:959px){

        /*表示当页面小于959px的时候执行它下面的CSS.*/
    } 

    /*针对限定的尺寸范围可以编写不同的样式：*/

    /*智能手机<480px*/
    @media(max-width:480px){}

    /*竖屏的平板<768px*/
    @media(max-width:768px) and (min-width:480px){}

    /*横屏的平板>768px*/
    @media(max-width:1024px) and (min-width:768px){}

    /*桌面PC>1024px*/
    @media(min-width:1024px){}

    /*高分屏图片优化：*/
    @media only screen and (-webkit-min-device-pixel-ratio: 1.5), 
    only screen and (min-device-pixel-ratio: 1.5) {}

</style>

4.灵活的flex布局： http://flexboxin5.com/
